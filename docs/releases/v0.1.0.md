## üéØ Release Overview

TSFulmen v0.1.0 establishes the **foundation** for the FulmenHQ TypeScript helper library ecosystem with complete repository structure, governance framework, and development infrastructure.

This release prioritizes:

- **Enterprise-grade governance**: Complete safety protocols and agent framework
- **Cross-language consistency**: Aligned structure with gofulmen and pyfulmen
- **Developer infrastructure**: Bun-based tooling, Makefile standard, quality gates
- **SSOT integration**: Crucible asset synchronization ready

This is a **structure release** - no application code is included. Core modules (config-path-api, crucible-shim, logging, foundry, schema-validation, three-layer-config, ssot-sync) will be implemented in v0.1.1+.

### SSOT Integration

Crucible SSOT asset synchronization via goneat:

**Configuration**: `.goneat/ssot-consumer.yaml`

```yaml
sync:
  sources:
    - path: ../crucible
      targets:
        - source: docs/standards
          dest: docs/crucible-ts/standards
        - source: schemas
          dest: schemas/crucible-ts
        - source: config
          dest: config/crucible-ts
```

**Synced Assets**:

- `docs/crucible-ts/` - Standards, architecture guides, SOPs
- `schemas/crucible-ts/` - JSON schemas for validation
- `config/crucible-ts/` - Default configurations
- `.crucible/metadata/` - Sync state and metadata

**Benefits**:

- Version-isolated, pinned assets in repository
- Offline operation capability
- Cross-language consistency
- Easy updates via `make sync-ssot`

## üöÄ Getting Started

### Installation

```bash

cd tsfulmen

# Bootstrap (install deps + tools)

# Sync assets from Crucible
```

## üìù Known Limitations

### Not Yet Implemented (Planned for v0.1.1+)

All 7 core modules per Fulmen Helper Library Standard:

1. **config-path-api**: XDG-compliant config directory resolution
2. **crucible-shim**: Typed access to embedded Crucible assets
3. **schema-validation**: JSON Schema validation with AJV
4. **three-layer-config**: Layered configuration loading
5. **foundry**: Pattern catalogs, HTTP status helpers, MIME types
6. **logging**: Progressive logging with policy enforcement
7. **ssot-sync**: Programmatic SSOT synchronization API

### Current Limitations

- **No Application Code**: This is a structure-only release
- **No npm Package**: Not yet published (pre-release)
- **Test Coverage**: Minimal (infrastructure only)
- **No Module Exports**: Only basic index.ts entry point
