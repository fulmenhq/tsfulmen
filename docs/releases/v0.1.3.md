## Pathfinder Module - Enterprise Filesystem Traversal

### Core Features

**Filesystem Discovery Engine:**

- Recursive directory scanning with glob pattern matching (`includePatterns`, `excludePatterns`)
- Streaming results via `findIterable()` for memory-efficient large directory processing
- `maxDepth` control for traversal depth limiting
- `followSymlinks` and `includeHidden` for advanced filesystem access
- Cross-platform path normalization and separator handling

**Ignore File Support:**

- `.fulmenignore` and `.gitignore` support with nested precedence rules
- Hierarchical ignore pattern loading from directory tree
- Child directory patterns override parent directory rules
- Configurable via `honorIgnoreFiles: boolean`

**FulHash Integration:**

- Optional streaming checksum calculation with `xxh3-128` (default) and `sha256` support
- Streaming implementation prevents memory exhaustion on large files
- Checksum errors handled gracefully with error metadata preservation
- Performance verified: `<10% overhead` with checksums enabled

**Security & Constraints:**

- Path traversal attack prevention with root boundary enforcement
- Three enforcement levels: `STRICT` (reject violations), `WARN` (log warnings), `PERMISSIVE` (no enforcement)
- `allowedPrefixes` and `forbiddenPatterns` for fine-grained access control
- Symlink resolution with escape detection

### Enterprise Observability

**Structured Error Handling:**

- All operations emit structured `FulmenError` on failure
- Correlation ID support for distributed tracing across operations
- Severity levels with appropriate categorization (`filesystem` domain)
- Schema-compliant JSON serialization for API consistency

**Telemetry Integration:**

- Automatic metric emission per TSFulmen taxonomy
- `pathfinder_find_ms` histogram for operation duration tracking
- `pathfinder_security_warnings` counter for constraint violations
- Integration with existing telemetry registry and export pipeline

**Progressive Logging:**

- Full integration with TSFulmen's progressive logging system
- Policy-driven logging profiles for different deployment scenarios
- Correlation ID propagation through all log entries
- Structured logging with domain-specific context

### Developer Experience

**Convenience Helpers:**

- `findConfigFiles()`: Discover YAML/JSON configuration files with default extensions
- `findSchemaFiles()`: Find `.schema.json` and `.schema.yaml` files using standard patterns
- `findByExtensions()`: Generic extension-based file discovery

**TypeScript Integration:**

- Full type safety with exported `PathfinderOptions` interface
- Integration with existing TSFulmen schema module for config validation
- Zero `any` types with strict mode compliance
- Comprehensive JSDoc documentation with usage examples

**API Design:**

- Progressive interface with sensible defaults
- Streaming support for large directory operations
- Configurable options with clear separation of concerns
- Backward-compatible API surface for future enhancements

## Migration Guide

**No migration required** for existing users. Pathfinder is a new module that can be adopted incrementally.

**For new Pathfinder usage:**

```typescript
import { Pathfinder } from "@fulmenhq/tsfulmen/pathfinder";

// Basic usage
const finder = new Pathfinder({
  root: "./src",
  includePatterns: ["**/*.ts"],
});

const results = await finder.find();
```
