# Release v0.3.0 (2026-06-06)

**Release Type**: Major Dependency Wave (breaking — Node engine floor)

## Overview

A coordinated wave that migrates the four major dependencies deferred from v0.2.10 — **archiver 8**, **pino 10**, **TypeScript 6** (dev), and **commander 15** — and **raises the Node engine floor to `>=22.12.0`** (was `>=20.0.0`).

The engine bump is the only breaking change; tsfulmen's own public exports are unchanged. No external consumers are affected (the galaxy runs Node 22+). The wave shipped as four independently reviewed PRs — archiver 8 (#6), pino 10 (#9), TypeScript 6 + Node 25 teardown fix (#10), and commander 15 + the engine floor (#11).

## What Changed

### Breaking

- **Node engine floor `>=20.0.0` → `>=22.12.0`** (`package.json` `engines.node`). Driven by commander 15, which is ESM-only and requires `node>=22.12.0` for `require(esm)`; the floor also subsumes pino 10's drop of Node 18. **Consumers must be on Node 22.12+.**

### Security

- **archiver 7.0.1 → 8.0.0** — escapes the **unpatchable lodash 4.x advisories** that archiver 7 pulled transitively via `archiver-utils`: `GHSA-r5fr-rjxr-66jc` (`_.template` code injection, **high**) plus two `_.unset`/`_.omit` prototype-pollution advisories (moderate). None have a fixed lodash 4.x, so dropping lodash was the only remedy — archiver 8 removes `archiver-utils`/`lodash` entirely. Its `readdir-glob → minimatch` chain resolves to a patched `minimatch@10.2.x`. Net `bun audit`: **23 → 17** findings (15 → 11 high).

### Changed

- **archiver 8 API migration** (`src/fulpack/core.ts`) — archiver 8 is a ground-up ESM rewrite that replaced the callable `archiver(format, options)` factory with format-specific classes. Migrated to `new TarArchive(...)` / `new ZipArchive(...)` and the imported `Archiver` type. **No public-API or behavior change** for fulpack (`create`/`extract`/`scan` unchanged).
- **Local archiver v8 type shim** (`src/fulpack/archiver.d.ts`) — archiver 8 ships no types and `@types/archiver` is frozen on the removed v7 factory API, so `@types/archiver` was dropped in favor of a focused ambient `declare module "archiver"` covering the v8 surface fulpack uses. The shim leaks no `archiver` types into the published `dist` surface.
- **pino 9.14.0 → 10.3.1** — clean major (no source changes). pino remains CJS with a default export, so the `src/logging/logger.ts` integration is unchanged. tsfulmen uses custom redaction (`logging/middleware.ts`), not pino's `redact`/`censor`; sinks are custom (not pino transports), so the thread-stream@4 / pino-abstract-transport@3 upgrades have no surface here.
- **TypeScript (dev) 5.9.3 → 6.0.3** — compiler bump, no `src/` changes. Added `ignoreDeprecations: "6.0"` to `tsconfig.json`: the `.d.ts` build (tsup 8.5.1) injects `baseUrl: "."` into the declaration compiler options, and TS 6 errors on the now-deprecated `baseUrl` (`TS5101`).
- **commander 14.0.3 → 15.0.0** — ESM-only and the driver of the Node floor bump. No source changes: tsfulmen is already ESM and the CLIs use the stable `import { Command }` API. The `schema export` command uses standalone negated flags (`--no-provenance`, `--no-validate`); commander 15's parsing change affects only paired positive+negative definitions, which tsfulmen does not use, so behavior is unchanged.

### Fixed

- **`create()` error path** (`src/fulpack/core.ts`) — the archive `error` handler threw synchronously inside the listener (surfacing as an uncaught exception) instead of rejecting `create()`. It now rejects with a wrapped `FulpackOperationError` across all three create paths.
- **Node 25 test teardown** — `src/config/__tests__/loader.test.ts` used the deprecated `fs.rmdir(path, { recursive: true })`, a hard failure on Node 25.x; switched to `fs.rm(path, { recursive: true, force: true })`.

### Tests

- **fulpack**: content round-trip (create → extract → compare bytes) for tar.gz/tar/zip incl. unicode + nested directories; `preserve_permissions` positives (exec-bit survives a tar.gz round-trip; tar mode reported via scan); a characterization test locking the documented ZIP unix-mode limitation (`scanZip` returns `"0644"` by design); and a `create()` error-path guard.
- **logging**: DEBUG/WARN severity-label assertions completing output coverage across all four logger methods.

## Consumer Impact

**Breaking: requires Node 22.12+.** tsfulmen's public API (`@fulmenhq/tsfulmen` and all subpaths) is otherwise unchanged — no code migration is needed beyond ensuring the runtime is Node 22.12 or newer. Consumers should upgrade for the archiver-8 security improvement and to align with the latest toolchain.

## Follow-ups

- Remove the `src/fulpack/archiver.d.ts` shim and restore `@types/archiver` once `@types/archiver@8` lands on DefinitelyTyped.
- Remove `ignoreDeprecations: "6.0"` once tsup stops injecting `baseUrl` (or before the TS 7 upgrade).
- Tracked: fulpack Tier 2 test hardening (#7), Go↔Node cross-impl fulpack parity corpus (#8), and CLI arg-parse testability (`import.meta.main` guard + extracted `buildProgram()`).

## Testing & Validation

- `make check-all` — typecheck, lint, **2131 tests passed** (16 skipped).
- `make build` + `bun run validate:all` — green against a freshly built `dist` (exports / tsup / source-modules / package / imports / types).
- `bun audit` — lodash highs cleared; **23 → 17** findings.
- CI green across all four PRs (`quality` ubuntu + macOS, `build`).
