# Release v0.3.2 (2026-06-17)

**Release Type**: Patch — app-identity follow-up

## Overview

A small follow-up to v0.3.1's app-identity work. Crucible **v0.4.14** added a formal `metadata.typescript` section to the app-identity schema (the npm analogue of `metadata.python`); v0.3.2 syncs tsfulmen's bundled Crucible SSOT to v0.4.14 and promotes `.fulmen/app.yaml` from its interim custom `metadata.console_scripts` field to that first-class section.

No code or behavior change. The Node engine floor is unchanged (`>=22.12.0`); the public API is unchanged. The only published delta is the refreshed bundled `crucible-ts` schemas.

## What Changed

### Changed

- **Synced Crucible SSOT `v0.4.13 → v0.4.14`** (`.goneat/ssot-consumer.yaml`). `make sync-ssot` refreshed the bundled app-identity assets under `schemas/crucible-ts`, `config/crucible-ts`, and `docs/crucible-ts`: the schema now carries the optional `metadata.typescript` object (`package_name` + `console_scripts: [{name, entry_point}]`, mapping to package.json `bin`), plus the `typescript-package` fixture and the `typescript_package` parity case.
- **Promoted `.fulmen/app.yaml` to `metadata.typescript`** — the three bins (`tsfulmen-schema`, `tsfulmen-signals`, `tsfulmen-prometheus`) move from the interim custom `metadata.console_scripts` field (added in v0.3.1 when no schema section existed) to the formal `metadata.typescript` section. `.fulmen/app.yaml` is the repo's own identity and is **not** in the npm `files` payload.

## Consumer Impact

**None functional.** No public API, runtime, or engine-floor change. Consumers that read tsfulmen's bundled Crucible app-identity schema (`schemas/crucible-ts/.../app-identity.schema.json`) will see the new optional `metadata.typescript` section.

## Testing & Validation

- `make validate-app-identity` — passes against the synced v0.4.14 schema (the formal `metadata.typescript` validates).
- `make check-all` — typecheck, lint, **2131 tests passed** (16 skipped); app-identity parity + schema-export parity pass, including the new `typescript_package` case.
- `make verify-artifacts` / `verify-local-install` / `verify-compile-smoke` — green.
