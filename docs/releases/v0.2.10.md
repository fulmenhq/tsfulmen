# Release v0.2.10 (2026-06-05)

**Release Type**: Security + Dependency + Infrastructure Maintenance

## Overview

A maintenance release with no public API changes. It clears a **CRITICAL** security advisory in the dev/test toolchain, syncs the Crucible SSOT to v0.4.13 (contract-parity role catalog), hardens CI, and brings the local toolchain up to date (goneat v0.5.13, normal commit/push/PR flow).

This release shipped as four reviewed PRs: process hardening (#1), Crucible v0.4.13 sync (#2), CI hardening (#3), and the dependency wave (#4).

## What Changed

### Security

- **vitest 4.0.18 → 4.1.8** — clears **CRITICAL** GHSA-5xrq-8626-4rwp (arbitrary file read/exec via the vitest UI server). `@vitest/coverage-v8` and `@vitest/ui` move in lockstep.
- Direct-dependency advisories also cleared by the wave: **ajv** 8.20.0 (GHSA-2g4f $data ReDoS), **picomatch** 4.0.4 (extglob ReDoS), **yaml** 2.9.0 (deep-nest stack overflow), **fastify** 5.8.5 (content-type/body bypass).
- Remaining `bun audit` findings are all transitive (archiver→lodash, vitest→vite/postcss, plus residual ajv/fast-uri/picomatch slots pulled by other deps) and are tracked for a follow-up wave alongside the deferred majors.

### Changed

- **Crucible SSOT → v0.4.13** (`ssot-consumer` ref v0.4.12 → v0.4.13): agentic role catalog `devlead`/`devrev`/`qa` → v1.0.1 (contract-parity focus), plus goneat v0.5.12 YAML-formatting alignment across the synced `config/crucible-ts` and `schemas/crucible-ts` assets.
- **Dependency wave (minor/patch — no majors)**: vitest / @vitest/coverage-v8 / @vitest/ui → 4.1.8; @biomejs/biome → 2.4.16 (+ `biome.json` `$schema`); tsx → 4.22.4; prettier → 3.8.3; @types/node → 25.9.1; @types/bun → 1.3.14; @types/picomatch → 4.0.3; fastify → 5.8.5; ajv → 8.20.0; picomatch → 4.0.4; tar-stream → 3.2.0; yaml → 2.9.0; pino → 9.14.0 (held on 9.x).
- **CI hardening**: `actions/download-artifact@v4 → @v4.1.3` (clears GHSA-cxww-7g56-2vh6 zip-slip), `oven-sh/setup-bun@v1 → @v2`, `BUN_VERSION 1.2.22 → 1.3.9` (aligned with the local toolchain), and **goneat pin v0.5.12 → v0.5.13** (picks up the `goneat format --check` YAML false-positive fix).
- **Process hardening**: removed the guardian browser-approval gate from the local git hooks (moved to the standard commit/push/PR flow with branch-protection / merge-policy controls); consolidated the CI goneat pin into a single `GONEAT_VERSION` workflow env; fixed `.goneat/dependencies.yaml` lockfile (`bun.lockb` → `bun.lock`).

### Fixed

- **picomatch options**: dropped the no-op `posixSlashes` option in `pathfinder/ignore.ts` — it was removed from `@types/picomatch` 4.0.3 and picomatch never honored it at runtime (zero behavior change; the code already normalizes backslashes before compiling the pattern).
- biome 2.4.16 import-ordering auto-fixes in `appidentity/loader.ts` and `fulencode/index.ts`.

## Consumer Impact

**No public API changes and no migration required.** All exports (`@fulmenhq/tsfulmen` and subpaths) are unchanged. Consumers should upgrade for the security fixes (notably the CRITICAL vitest advisory in the dev/test toolchain) and the refreshed, contract-parity Crucible role catalog.
