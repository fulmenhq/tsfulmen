# Release v0.2.8 (2026-02-20)

**Release Type**: Feature + Security + Dependency Maintenance

## Overview

Adds the typed role catalog API for programmatic access to Crucible agentic role prompts. Patches three security advisories in direct dependencies. Bumps 13 dependencies including TypeScript 5.9.3 and Biome 2.4.3.

## What Changed

### Added

- **Typed Role Catalog API** (`@fulmenhq/tsfulmen/crucible`)
  - `listRoleSlugs()` — returns sorted role slugs (README excluded)
  - `loadRole(slug)` — loads and parses a single role YAML into a fully typed `RolePrompt`
  - `loadRoleCatalog()` — loads all roles into a `ReadonlyMap<string, RolePrompt>`
  - 6 exported types: `RolePrompt`, `RoleMindset`, `RoleEscalation`, `RoleExample`, `RoleRequiredReading`, `RoleRequiredReadingFile`
  - Throws `AssetNotFoundError` with fuzzy-match suggestions on invalid slugs
  - Slug validation: `^[a-z][a-z0-9]*$` (Crucible v0.4.12 convention)
  - 13 invariant-based tests (avoids brittleness when roles are added/removed)

- **Role Catalog Documentation** — comprehensive section in `docs/guides/crucible-assets.md`
  - Quick start, type reference, error handling patterns
  - Migration guide for repos previously doing manual YAML parsing (e.g., brooklyn-mcp)


### New API usage

```typescript
import {
  listRoleSlugs,
  loadRole,
  loadRoleCatalog,
} from "@fulmenhq/tsfulmen/crucible";
import type { RolePrompt } from "@fulmenhq/tsfulmen/crucible";

const slugs = await listRoleSlugs();
const role = await loadRole("devlead");
const catalog = await loadRoleCatalog();
```

### For repos previously parsing role YAMLs manually

See the migration guide in `docs/guides/crucible-assets.md` — replace ad-hoc YAML loading with the typed API above.
