# Release v0.4.1 (2026-06-26)

**Release Type**: Patch — asset-resolver security hardening + CI tooling

## Overview

A small follow-up to v0.4.0's compile-safe asset embedding: hardens the public
`@fulmenhq/tsfulmen/assets` surface (secrev P3 follow-ups) and fixes the goneat format
gate so Markdown drift is caught before it ships. No public API removals; the only
behavior change is rejecting inputs that were never valid assets. Node engine floor
unchanged (`>=22.12.0`).

## What Changed

### Security

- **`AssetResolver` namespace tightened to the shipped subtree.** Logical paths and glob
  patterns on the package's own resolver must resolve under `<namespace>/crucible-ts/`
  (the only subtree the package ships) — not merely a `schemas`/`config`/`docs` top
  segment. This matches the real published surface; out-of-subtree inputs were always
  not-found, now they're rejected up front. Consumer-supplied `baseDir` overrides
  (`enforceNamespace: false`) keep their relaxed namespace (traversal guards still apply).
- **`FsAssetResolver` rejects symlink escapes.** `read` and `has` now resolve symlinks
  (`realpath`) and verify the target stays within `baseDir`, so a symlink slipped into the
  asset tree cannot read outside the asset root. `list` already used
  `followSymbolicLinks: false`.

### Changed (CI / tooling)

- **goneat format gate now covers Markdown.** `.goneat/assess.yaml` gained a `format:`
  block; without it goneat's format category fell back to whitespace/EOF-only and never
  scoped Markdown/JSON/YAML — so the format leg of the pre-commit/pre-push hooks and
  `goneat assess --categories format` silently skipped Markdown (this let trailing-
  whitespace drift slip past every gate during v0.4.0).
- **`.goneatignore` expanded** to exclude the Crucible-synced SSOT trees
  (`config|schemas|docs/crucible-ts`), generated modules (`src/assets/generated`), test
  fixtures (`**/fixtures`, `**/__fixtures__`), and build output — so `goneat format`
  (which honors `.goneatignore`) doesn't reformat upstream-owned or generated content.
- **`make fmt` fixed.** It ran `bash -c '$$GONEAT format …'` with an unexported
  `$GONEAT`, so the subshell ran `format` bare (`command not found`) and silently skipped
  all goneat formatting. Now runs goneat directly in the recipe shell.

## Consumer Impact

- **Additive/hardening only.** The new namespace restriction and symlink check reject
  inputs that were never valid package assets; normal usage (resolving real
  `crucible-ts` assets) is unchanged. CI/tooling changes do not affect the published
  package or runtime.

## Testing & Validation

- `make check-all` — now Markdown-aware; full suite + parities + embedded drift.
- New hardening tests: namespace-tightening (accept `<ns>/crucible-ts/…`, reject
  in-namespace-but-not-crucible-ts) and symlink-escape rejection in `FsAssetResolver`.
- Proof the gate works: injecting trailing whitespace into a doc now fails
  `goneat assess --categories format` (previously passed).
- Pre-tag gauntlet: `make build`, `verify-artifacts`, `verify-local-install`,
  `verify-compile-smoke`, `verify-embedded-compile`.
